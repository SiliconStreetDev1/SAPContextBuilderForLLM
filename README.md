# SAP Context Builder

A desktop utility for SAP Architects and ABAP Developers. It compiles local and remote (GitHub) SAP repositories into XML chunks formatted for LLM context windows.

## Why Create Context?

Large Language Models (LLMs) operate within strict token limits and have no inherent visibility into your specific SAP environment. Feeding an LLM a single isolated file often results in code that references non-existent variables, hallucinates method calls, or breaks architectural patterns.

Creating a compressed context payload solves this:
* **Improves Hit Rate:** By stripping whitespace, removing carriage returns, and excluding irrelevant frontend/build directories, you maximize the amount of relevant codebase (Data Dictionary, Global Classes, CDS views) that fits into a single prompt.
* **Reduces Hallucinations:** Providing the broader project architecture allows the LLM to read existing definitions rather than guessing them.
* **Enforces Standards:** Packaging system instructions alongside the raw code guarantees the LLM adheres to strict rules (e.g., forcing multi-line global class definitions or prohibiting Hungarian notation) before it generates a response.
* Read-Only by Design: The SAP Context Builder only reads your repository files to generate the XML output. It never modifies, deletes, or uploads your source code.

## ✨ Core Features

* **Multi-Repository Aggregation:** Compile context from multiple local paths and remote GitHub repositories simultaneously. Fully supports both backend ABAP and frontend SAP UI5 / Fiori codebases in a single payload.
* **Maximum Payload Compression:** Removes carriage returns and allows strict directory exclusion to aggressively compress payloads, ensuring the highest possible hit rate within LLM token limits.
* **ABAP Standards:** Includes a system prompt instructing the LLM to output clean, modular, and well-commented ABAP without Hungarian notation.
* **OO Syntax Rules:** Enforces standard global class definitions 
* **Profile Management:** Saves project configurations (sources, extensions, prompts) to Local Storage.
* **Secure GitHub Integration:** Retrieves private repositories using a Personal Access Token (PAT).

### 🌐 Distributed Development Unification
The SAP Context Builder bridges the gap between distributed developments by linking multiple repositories into a single, cohesive AI context. 

* **Unified Context Stream**: Aggregate disparate components—such as ABAP backends, UI5 frontends, and shared libraries—into one synchronized XML payload.
* **Multi-Source Ingestion**: Allows you to mix local file system paths and remote GitHub URLs within the same session.
* **Holistic Architecture**: By consolidating separate codebases, the tool provides the AI with a complete architectural view of your entire project landscape, rather than isolated fragments.
* **Seamless Aggregation**: The background engine iterates through every configured source, cloning remote repositories into isolated temporary directories and merging all artifacts into a centralized collection before intelligent chunking. 

### Remote Source Handling (Git)
Repositories clone into unique, timestamped folders within the system temp directory. Shallow clones (`--depth 1`) pull only the latest code to stay fast and minimize disk usage. The temporary folder is automatically deleted once the files are ingested into memory.

**Safety Guards:**
* Root scans are blocked to prevent accidental system-wide loops.
* 15-level recursion limit on all directory reads.
  
### Prerequisites
* Node.js (v16 or higher)

1. Clone or download this repository.
2. Install dependencies by running: `npm install`
3. Start the local server by running: `npm start`
4. Navigate to `http://localhost:3000` in your web browser.
## ⚙️ Configuration (`config.json`)

The `config.json` file in the root directory controls the engine's compilation limits, exclusions, and UI toggles. 

### Key Parameters:

* `maxCharsPerChunk`: The mathematical ceiling for each output chunk. Lower this if your target LLM is rejecting the payload size.
* `maxFileSizeBytes`: Skips individual files larger than this limit to prevent memory crashes.
* `ignoreDirs`: **Crucial for hit-rate optimization.** Add the folder names of heavy BSP applications, Fiori `webapp` folders, or node modules here to entirely bypass them during the build process.
* `ignoreFiles`: Specific filenames to skip across all directories.
* `uiExtensionGroups`: Defines the categories and specific file extensions (.clas.abap, .asddls, .xml) available to whitelist in the UI.

* ## 🛠️ Usage Workflow

1.  **Select a Profile:** Use the dropdown to select a saved project or create a new one using "Save As".
2.  **Add Repositories:** Add local directory paths or GitHub URLs. If using GitHub, enter your Personal Access Token (PAT) and save it to your OS Keychain.
3.  **Filter Artifacts:** Check the boxes for the specific file types you need (e.g., Classes, CDS Views, Dictionary XMLs).
4.  **Refine the Prompt:** Modify the Master AI Prompt if you need to add specific context about your current sprint or ticket.
5.  **Compile:** Click **📦 1. COMPILE FULL CONTEXT**. The engine will clone, read, strip, and pack the data into memory.
6.  **Export:** Use the generated **COPY** buttons to paste data manually, or click **Download XML** to save the full payload (which you can also copy entirely into your LLM as file depending on LLM used ).

## 🔒 Token Security & Risk Mitigation

The SAP Context Builder utilizes a proprietary, zero-dependency security architecture. It eliminates the risks associated with abandoned third-party libraries.

### Cryptographic Architecture: AES-256-GCM
Credential storage is managed via an **Authenticated Encryption Vault** that adheres to modern cryptographic standards:

1. **Machine-Level Cryptographic Binding:** The encryption key is dynamically derived using the `scrypt` key derivation function (KDF), salted with a combination of the **OS UID** and the **Hardware Hostname**. This creates a unique machine-bound identity; the vault is logically locked to the physical device and cannot be decrypted if moved to external hardware.

2. **Authenticated Encryption (AEAD):** The system utilizes `aes-256-gcm`. Unlike standard AES, the Galois/Counter Mode (GCM) provides an **Authentication Tag**. This ensures the integrity of the ciphertext; if a single bit of the vault is altered by a background process or malware, the cryptographic verification will fail, and the system will prohibit decryption.

3. **OS-Level Least Privilege Access:** The encrypted payload (`vault.enc`) is isolated in a restricted directory (`~/.sap-context-builder/`). It is committed to the disk using **Strict 0o600 POSIX Permissions** (Read/Write for the owner ONLY). This enforces OS-level isolation, blocking access from other users or unauthorized service accounts on the system.

4. **Zero-Dependency Native Backend:** To mitigate Supply Chain Attacks, the security module is written in pure JavaScript, interfacing directly with Node’s native `crypto` module. This leverages the **OpenSSL 3.0+ backend** statically linked to the Node runtime, ensuring the most robust, audited, and up-to-date cryptographic primitives available.

### Token Lifecycle
Tokens are decrypted into volatile memory **only** when active API calls require them and are never logged, cached to disk in plaintext, or transmitted to any third party besides the targeted LLM or GitHub endpoints.

## 🔐 Privacy & Data Handling

Since this tool involves sending your codebase to external LLMs, you should be mindful of how that data is stored and used for training. 

### Recommended Privacy Settings
If you are working with proprietary enterprise code, it is highly recommended to disable activity tracking/training in your LLM's settings:

* **Google Gemini:** Go to [myactivity.google.com](https://myactivity.google.com), find **Gemini Apps Activity**, and toggle it to **Off**. This prevents your prompts and code context from being used to improve Google's models.
* **ChatGPT:** Go to **Settings** > **Data Controls** and disable **Chat History & Training**.
* **Claude:** Anthropic generally does not use Pro/Team/Enterprise data for training by default, but always double-check your specific plan's privacy terms.

**Note:**Disabling these settings is a standard "best practice" for developers using web-based LLMs for professional work. Even with these settings off, always follow your organization's specific AI usage policies.
